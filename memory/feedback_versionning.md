name: Toujours versionner les wireframes
description: Avant toute évolution significative, sauvegarder une copie versionée (V1, V2...) du fichier de travail

Avant chaque étape importante, créer une copie du fichier de travail avec le suffixe de version : `nom-V1.html`, `nom-V2.html`, etc. Le fichier versionné ne doit jamais être modifié — seul le fichier de travail (sans suffixe) évolue.

**Why:** L'utilisateur veut pouvoir revenir à une version précédente à tout moment si quelque chose gêne dans l'évolution du design.

**How to apply:** Dès qu'une version est validée ou qu'on s'apprête à faire un changement significatif, faire un `cp fichier.html fichier-VN.html` avant de continuer. Toujours mentionner la version sauvegardée à l'utilisateur.
